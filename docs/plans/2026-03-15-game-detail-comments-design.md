# Game Detail + Comments — Design

## Цель

Страница игры `/games/:id` с полной информацией, кнопкой «Назад», счётчиком просмотров и блоком комментариев (дерево, лайки/дизлайки, ответы).

## Решения

### Маршрутизация

- **`/games/:id`** — страница игры (id = user_game.id)
- **`/@tag`** — профиль пользователя по tag (для кнопки «Назад» на чужую игру)

### Страница игры

**Layout:**
- Кнопка «Назад» → `/` (своя) или `/@ownerTag` (чужая)
- Блок игры: картинка, название, жанры, разработчик, издатель, release_date, стейт, избранное, заметка, даты, часы
- **Счётчик просмотров** — отображается на странице
- Блок комментариев ниже

**Просмотры:**
- Поле `view_count` в UserGame (integer, default 0)
- Инкремент при `GET /games/:id` (один раз за сессию/период на пользователя — опционально; MVP: инкремент при каждом запросе)

### Блок комментариев

**Модель:**
- `GameComment`: id, game_id, user_id, parent_id (null для корня), text (max 200), created_at
- `GameCommentVote`: comment_id, user_id, is_like (bool) — пользователь может поставить и лайк, и дизлайк на одно сообщение

**API:**
- `GET /games/:id/comments` — дерево комментариев
- `POST /games/:id/comments` — создать (text, parent_id?)
- `POST /games/:id/comments/:commentId/vote` — лайк/дизлайк (is_like: bool)

**Ответ комментария:**
- id, text, created_at, author (username, tag, avatar_url), like_count, dislike_count, current_user_voted (liked?, disliked?), children

**UI:**
- Карточка: аватар, username, @tag, текст, дефолтные эмодзи, лайк/дизлайк со счётчиками, «Ответить»
- Дерево: ответы с отступом; при > 3 ответах — отступ ml-4/ml-6
- Поле ввода: textarea max 200, «Отправить»

**Валидация:** text 1–200 символов.

### Backend

**GET /games/:id** — публичный, возвращает игру + owner (username, tag, avatar_url) + view_count. Инкремент view_count.

**GET /users/by-tag/:tag** или **GET /profiles/:tag** — профиль + игры (для страницы /@tag).

**Права:**
- GET /games/:id — публичный
- Комментарии, голоса — только авторизованные

### Ошибки

- 404 — игра не найдена
- 401 — комментарии/голоса без авторизации
- 400 — текст пустой или > 200 символов

### Граничные случаи

- Удалённый пользователь — «Удалённый пользователь»
- Глубина дерева — ограничить 5 уровней
- Эмодзи — фиксированный набор, вставка в текст по клику
